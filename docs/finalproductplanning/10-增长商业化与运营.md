# 10-增长商业化与运营

本文整合 yunmao 的增长策略、商业化路径、运营 Playbook、动物福利与公益约束。所有玩法必须满足 00 第 8 节非目标与 06 第 6 节合规要求；如有冲突以 00/06 为准。

## 1. 增长方法论

- **不靠刺激性玩法换增长**：拒绝“无限喂”“大胃王”等可能引发动物福利争议的机制。
- **真实陪伴优于一次性爆款**：内容、关注、提醒、回访构成主线，分享是结果而非起点。
- **每个增长动作可观测**：所有渠道、活动、玩法必须可归因，量化漏斗，及时止损。

## 2. 增长漏斗

```text
认知  → 触达
访问  → 进入房间 (Web/H5/App)
首播  → 完成首次播放
首喂  → 完成首次投喂
关注  → 关注 ≥ 1 只猫
回访  → 7 日内回访 / 通知点击
留存  → 次日/7 日/30 日留存
付费  → 完成首次付费 (后期)
推荐  → 分享/邀请 (后期)
```

每个环节定义指标与责任团队（参见 11 章）。

## 3. 渠道矩阵

| 渠道 | 内容形态 | 关键 KPI | 一期投入 |
| --- | --- | --- | --- |
| 微信公众号 | 软文 + 推送 | 关注数 / 点击率 / 跳转转化 | 内容 1 人兼职 |
| 微信视频号 / 视频号直播 | 短视频 + 直播切片 | 播放 / 互动 / 引流注册 | 1 人 |
| 小红书 | 图文 / 短视频 | 收藏 / 点赞 / 评论 / 关键词搜索 | 1 人 + KOC 合作 |
| 抖音 / 快手 | 短视频 + 话题 | 播放 / 完播 / 评论 / 引流 | 1 人 + KOL 合作（按预算）|
| B 站 | 中长内容 | 三连 / 关注 / 引流 | 视情况投入 |
| 微博 | 话题 / 转发 | 话题阅读量 / 讨论数 | 仅大事件造势 |
| 行业媒体 / 公众号 | 投放或对外报道 | 引流 / 媒体覆盖 | 不定 |
| Web SEO | 房间分享页、猫咪主页、平台介绍 | 自然流量 | 持续优化 |
| 线下 | 救助站合作、猫咖活动 | 拉新 / PR | Phase 2 起 |

策略：

- 内容运营“给猫咪故事，不卖产品”。
- 投放前先把首屏体验和投喂闭环做稳，避免“流量来了体验崩了”。
- 反作弊从首发开始：邀请关系绑定、设备指纹、行为序列分析。

## 4. 关键增长机制

### 4.1 通知触达

- 关注的猫开播 / 可投喂 / 投喂结果 / 设备异常 / 房间公告。
- 频控：单用户单日 ≤ N 条非互动通知（默认 N=3，可配置）。
- 全局静默时段：默认 22:30–08:00 不发非紧急通知。
- App 推送失败 → 站内消息保底；不重复二次推送。

### 4.2 邀请有礼

- 双向奖励：邀请者与被邀请者均获积分/免费投喂券。
- 反作弊：
  - 同一设备指纹不可被多个邀请同时绑定
  - 邀请关系 30 天有效，超期作废
  - 新账号需完成首喂（且健康投喂时间内）才计入有效邀请
- 邀请上限：单账号 30 天内有效邀请人数封顶

### 4.3 任务体系

- 一期：观看 ≥ 1 房间、关注 ≥ 1 只猫、首次投喂、分享 1 次。
- 不引导用户“连续 N 天投喂”，避免压力催投。
- 奖励聚焦虚拟权益和体验型（专属徽章、海报模板）。

### 4.4 分享与海报

- 投喂成功后展示“可分享”入口（轻量，不阻断主流程）。
- 海报内容：猫咪截图 + 用户昵称（脱敏可选）+ 投喂时间 + 小程序码 / 短链。
- 用户可关闭“分享时显示昵称”。

## 5. 商业化策略

### 5.1 收入分层

| 阶段 | 主要收入 | 说明 |
| --- | --- | --- |
| Phase 1 | 几乎不计收入；可接受捐赠/赞助 | 重在闭环验证 |
| Phase 2 | 软商业化：品牌房间、机构合作 | 不开放面向 C 端的高价付费 |
| Phase 3 | 投喂券 / 会员 / 公益赞助至少一种 | 真实扣款，主链路 |
| Phase 4+ | 多元化：电商、机构 SaaS、品牌联名、广告 | 在保证体验前提下扩展 |

### 5.2 候选商业化产品

| 产品 | 形态 | 价格区间（参考） | 价值主张 | 风险 |
| --- | --- | --- | --- | --- |
| 投喂券 | 单次/套餐 | 1–10 元单次 | 高级猫粮、限定特效、加速冷却（限度） | 不可绕过动物福利上限 |
| 月卡/年卡会员 | 订阅 | 月 19–39 元，年 199–299 元 | 专属称号、优先排队、海报模板、夜间频道等 | 必须可随时退订 |
| 公益赞助 | 一次 / 定额 | 自定义 | 救助站房间，公开物资使用记录 | 必须与合规机构合作 |
| 机构 SaaS | 年费 | 千元至万元级 | 多设备/多房间管理、数据看板 | 客户支持成本 |
| 品牌联名/广告 | 房间装饰、活动 | 商务议价 | 品牌曝光、定向人群 | 不能影响主播体验 |
| 周边商品 | 实物 | 几十至上百 | 品牌延伸 | 库存与售后 |

### 5.3 端差异化（与 03 第 9 节呼应）

- **iOS**：虚拟权益必须 IAP，App 内不能提及外部支付。
- **Android / Web / H5**：微信支付、支付宝优先；公益赞助必须接合规通道。
- **会员权益跨端一致**：iOS IAP 与外部购买的权益在服务端归一为同一权益，避免端内套利。

### 5.4 动物福利约束（硬约束）

- 任何付费产品不可“突破每日上限” / “解锁额外出粮”；最多缩短个人冷却但不能绕过房间冷却与上限。
- 排行榜不展示“吃得最多”的猫；展示“健康陪伴时长”“被关注度”等正向指标。
- 救助站房间禁用任何竞速类玩法。
- 一旦出现舆论事件，运营有权一键关闭付费投喂，仅保留免费/赞助路径。

## 6. 运营 Playbook

### 6.1 房间入驻流程

```text
申请 (Web/App 表单)
  → 资质审核 (实名/猫咪健康/设备)
  → 设备发放/绑定 (运营协助)
  → 测试房间 (内测白名单)
  → 正式开播 (按内容审核结果开启投喂)
  → 持续巡检 + 月度回访
```

每一步运营有 SLA（参见 06 第 13 节客服支持）。

### 6.2 内容审核

- 弹幕：先拦后审；敏感词命中立即拦截；高风险用户灰度审核。
- 画面：定时抽帧 + 第三方内容安全 API；命中后人工二次复核。
- 举报：房间页一键举报；30 分钟内首响应，4 小时内闭环。

### 6.3 设备巡检

- 每日：在线率、卡粮率、粮量阈值。
- 每周：设备版本分布、OTA 进度、机构反馈。
- 每月：硬件总成本、损坏率、合作方对账。

### 6.4 数据值守

- 每日早会过北极星指标（00 第 7 节 + 06 第 9 节）。
- 异常波动 > 阈值 → 自动 alert + 值班人确认。

### 6.5 危机沟通

- 模板：站内公告、App 弹窗、Web banner、外部声明。
- 涉及动物福利或公益的争议必须由产品 + 法务 + 运营三方联合发声。

## 7. 用户分层与生命周期

| 阶段 | 定义 | 干预策略 |
| --- | --- | --- |
| 新用户 | 注册 ≤ 7 天 | 新手任务、首喂引导、推送频率克制 |
| 活跃用户 | 7 日内 ≥ 3 次活跃 | 关注推荐、海报激励 |
| 沉默用户 | 30 日未活跃 | 1 次召回推送 + 公众号软文；不再轮番打扰 |
| 流失用户 | 90 日未活跃 | 不主动推送；保留账号 |
| 高价值用户 | 持续付费/分享 | VIP 称号、内测白名单、社群运营 |
| 风险用户 | 风控命中 | 限制投喂/弹幕，必要时封禁 |

## 8. 反作弊与风控

- 设备指纹 + 行为序列 + 异常时空判定。
- 高价值动作（投喂、邀请、付费）必须经过风控决策。
- 风控阈值通过配置中心调整，灰度发布。
- 误杀回滚：人工复核通道 + 误杀率 SLA。

## 9. 公益策略

- 公益必须可证明：合作机构资质公开、资金/物资流向公开、定期报告。
- 不允许平台“代为承诺”公益捐赠数量；只承诺“将赞助按比例转交合作机构”。
- 救助站房间默认免费投喂为主，赞助可叠加；不强制要求救助站配合营销活动。

## 10. 数据驱动机制

- 每个新机制上线必须定义“成功指标”和“失败回滚条件”。
- A/B 测试：从 Phase 2 起建立；样本量、显著性、运行周期标准化。
- 实验治理：每季度复盘已上线实验，不留长期未结论的实验代码。

## 11. 与早期文档的差异

`docs/earlyproductplanning/` 中提到的若干玩法本文件不采纳或延期：

| 早期玩法 | 处理 | 原因 |
| --- | --- | --- |
| “大胃王”/“最贪吃”排行榜 | 不采纳 | 动物福利风险 |
| “虚拟工会”PvP | 暂缓 | 易引发刷投喂 |
| 远程激光笔/逗猫棒 | 暂缓 | 安全 + 动物福利 |
| AI 猫语翻译 | 暂缓 | 与主线无关 |
| 猫咪售卖 / 寄养 | 暂缓 | 资质与合规复杂 |
| 高竞争打赏排行榜 | 简化 | 健康榜替代 |
| 自营/联营电商 | 延期 | Phase 3+ 评估 |
