# 11-数据指标与术语表

本文汇总跨文档使用的核心指标、事件字典、术语与缩写。所有指标定义以本文为准；如其他文档出现差异，以本文为正并同步修正。

## 1. 北极星与一级业务指标

| 指标 | 定义 | 责任 |
| --- | --- | --- |
| 日活跃用户 DAU | 自然日内有效会话（≥ 1 次房间停留或投喂/弹幕）的去重用户数 | 增长 |
| 投喂端到端成功率 | 设备实际出粮成功数 / 服务端 accepted 数 | 产品 + 后端 |
| 首屏播放成功率 | 首帧成功 < 1.5s P95 的会话数 / 房间进入总数 | 媒体 |
| 直播卡顿率 | 卡顿总时长 / 播放总时长（按会话维度加权） | 媒体 |
| 通知触达率 | 30s 内任一通道送达 / 应送达 | 通知/Ops |
| 7 日留存 | 注册 7 日内仍活跃用户 / 注册数 | 增长 |
| 客诉率 | 客服工单 / DAU（万分比） | 客服/产品 |

## 2. 二级业务指标

### 2.1 用户与漏斗

- 注册成功率：注册成功 / 验证码请求
- 登录成功率：登录成功 / 登录请求
- 首次进入房间率：进入房间 / 首次访问
- 首次投喂率：完成首投 / 注册用户
- 关注率：关注 ≥ 1 只猫 / 进入过房间用户

### 2.2 房间与媒体

- 房间在线率：在线时长 / 总自然时长
- 推流重连次数：按房间日聚合
- 协议降级次数：按 CDN/协议维度
- 投喂叠加延迟：设备回执到画面叠加 P50/P95
- App 后台恢复成功率：后台 → 前台续播成功 / 切换次数

### 2.3 设备

- MQTT 在线率：在线设备数 / 全部设备数
- 设备指令 ack 延迟 P95
- 卡粮率：卡粮事件 / 出粮指令
- 粮量不足率
- 摄像头与投喂机一致率：双在线/任一在线

### 2.4 内容/弹幕

- 弹幕拦截率：被审核拦截 / 总发送
- 举报响应时长 P95
- 违规事件数（按等级）

### 2.5 商业（Phase 3+）

- 付费用户数 / 付费率 / ARPU / ARPPU
- 订单成功率 / 支付回调延迟
- 退款/补偿数及金额

## 3. 客户端 QoE 指标（详见 02 第 12 节）

- 首帧时间 `time_to_first_frame_ms`
- 卡顿次数 `stall_count`、卡顿总时长 `stall_total_ms`
- 平均码率 `avg_bitrate_bps`
- 切档次数 `profile_switch_count`
- 协议回退 `protocol_fallback_count`
- 退出原因 `exit_reason`
- 后台恢复成功 `bg_resume_success`
- 投喂叠加延迟 `feed_overlay_delay_ms`

## 5. 术语表

| 术语 | 含义 |
| --- | --- |
| 房间 (room) | 一个直播单元，含一个或多个摄像头、设备和一只或一组猫咪 |
| 猫咪 (cat) | 平台档案化的具体猫咪个体，可关联至多个房间 |
| 投喂请求 (feed_request) | 用户发起的一次投喂意图，是用户视角的幂等单位 |
| 设备指令 (device_command) | 平台向设备下发的一次出粮命令，是设备视角的幂等单位 |
| 冷却 (cooldown) | 个人 + 房间两级，防止过度投喂的时间窗约束 |
| 禁投喂时间窗 (no-feed window) | 房间配置的不允许投喂的时段（如夜间） |
| 健康上限 (health limit) | 每只猫每日主食/零食的最大额度 |
| 多档播放 (ABR profile) | 同一直播的多种码率/分辨率组合 |
| LL-HLS | Low Latency HLS，低延迟版 HLS |
| SFU | Selective Forwarding Unit，WebRTC 转发服务 |
| QoE | Quality of Experience，端到端体验质量指标 |
| QoS | Quality of Service，网络层质量指标 |
| 边缘代理 (edge agent) | 部署在救助站/猫咖等现场的本地代理，缓存指令并代理设备 |
| 媒体边缘 (media edge) | 部署在区域 PoP 的媒体接入/分发节点 |
| 控制面 / 数据面 | 控制面承载业务、配置、编排；数据面承载媒体/事件流量 |
| 契约仓库 (contracts repo) | 集中维护 OpenAPI / Protobuf / 错误码 / Token 的单仓 |
| 灰度 / 金丝雀 | 按比例放量的发布策略 |
| Runbook | 应急处置手册 |
| ADR | Architecture Decision Record，架构决策记录 |
| 北极星指标 | 反映核心业务健康度的最高优先级指标 |
| 错误预算 | (1 - SLO) 内允许的故障量；耗尽即暂停高风险变更 |
