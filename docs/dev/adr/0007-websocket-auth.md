# ADR-0007 WebSocket 网关鉴权

- 状态：Proposed（架构负责人建议）
- 日期：2026-05-24
- 关联：`01-技术选型与系统架构.md`、`04-设备接入数据模型与API边界.md`

## 决策

推荐由 **user-svc 签发登录 JWT**，由 **room-svc 签发短期 room subscription token**。WebSocket 网关校验 token 后允许订阅；游客只允许只读订阅公开视频与有限事件，投喂、弹幕、关注、支付等写操作必须登录。

## 理由

- 登录身份与房间订阅权限分离，避免长期用户 JWT 直接携带复杂房间权限。
- room token 可短 TTL、按房间、按协议、按清晰度和游客身份下发，便于未来 CDN/播放鉴权复用。
- 游客只读能保留分享转化路径，但不会放大互动、风控和内容审核风险。

## 替代方案

- 网关直接验 user JWT 并查询房间权限：实现简单，但耦合业务服务且长连建立路径变重。
- 完全禁止游客：安全简单，但损失首访体验。
- 完全允许匿名订阅和发言：早期演示方便，但内容安全和滥用风险不可接受。

## 重新评估条件

- API Gateway/边缘鉴权组件统一承担 JWT 与 room token 校验。
- 产品决定第一版完全封闭内测，不开放游客访问。
- 直播播放 URL、WS 订阅、弹幕写入需要统一成一套边缘 token。
