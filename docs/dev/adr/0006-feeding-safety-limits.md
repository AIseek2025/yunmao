# ADR-0006 投喂安全默认值

- 状态：Proposed（架构负责人建议，需业务/兽医最终确认）
- 日期：2026-05-24
- 关联：`04-设备接入数据模型与API边界.md`、`09-测试质量与发布工程.md`

## 决策

MVP 默认值推荐更保守：`room_cooldown=30s`、`per_user_room_cooldown=60s`、`per_cat_daily=12g`。这些值必须进入后台可配置策略，并支持按房间、猫、设备型号覆盖。

## 理由

- 动物福利与舆论风险优先于互动增长，默认值应偏保守。
- 当前实现与文档不一致：代码为 `15s/60s/100g`，交付报告写 `15s/30s/20`，必须收敛到单一配置源。
- 不同投喂机出粮精度、猫只体重、健康状态差异很大，硬编码默认值不能长期存在。

## 替代方案

- 保持当前 `15s/60s/100g`：演示更顺滑，但不适合作为默认安全策略。
- 按运营活动临时放宽：可以支持，但必须走后台审批和审计。
- 只做前端限流：不可接受，无法防止重复请求、脚本或多端并发。

## 重新评估条件

- 兽医或运营给出更精确的猫只画像、粮种热量、设备出粮误差和日粮建议。
- MVP 真实房间反馈表明默认值过严导致体验不可用。
- 后台策略、Redis 原子限流和审计链路完成后，可按房间灰度放宽。
