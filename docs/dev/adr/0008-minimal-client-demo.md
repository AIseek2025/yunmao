# ADR-0008 最小客户端演示

- 状态：Proposed（架构负责人建议）
- 日期：2026-05-24
- 关联：`08-多端工程组织与协作.md`、`09-测试质量与发布工程.md`

## 决策

推荐立即在 `clients/web-demo` 下添加最小 Web demo：flv.js 或 mpegts.js 播放 HTTP-FLV、WebSocket 订阅房间事件、按钮触发投喂请求。该 demo 只作为联调和回归工具，不作为正式 Web/H5 前端。

## 理由

- 当前底座已经有 media-edge、gateway、feeding-svc、device-edge，缺少一个可重复打开的端到端观测面。
- curl + websocat 对开发者可用，但对产品、QA、运营和回归验证不友好。
- 最小 demo 能尽早暴露协议、端口、CORS、播放兼容性和事件时序问题。

## 替代方案

- 暂不做客户端：节省半天到一天工作量，但端到端回归仍依赖手工命令。
- 直接启动正式前端工程：范围过大，会混入设计系统、路由、登录态等非底座问题。
- 用 Postman/Insomnia 集合：适合 API，不适合直播播放与 WS 事件。

## 重新评估条件

- 正式 Web/H5 工程已启动，并能覆盖同等 PoC 回归能力。
- 播放策略转向云厂商 SDK，demo 需要同步改为 SDK 验证页。
- 端到端自动化测试已经覆盖播放、WS、投喂闭环。
